# Create and use plug-ins in Android

You can use plug-ins to deliver resources and call Java and C++ code created outside of Unity from your C# **scripts**. They enable you to access features, like third-party code libraries and operating systems calls, that would otherwise not be available to Unity. For more information about plug-ins with Unity, see [Plug-ins](./plug-ins.md).

This section describes how to create your own plug-ins and use them in Android projects. The information on these pages assumes you already know how to create native plug-ins for Unity. For more information about native plug-ins and their uses, see [Native plug-ins](plug-ins-native.md).

## Unity as a Library

If you upgrade your project to Unity 2019.4 or above, the introduction of Unity as a Library might require you to adapt your [native](plug-ins-native.md) and [managed](plug-ins-managed.md) plug-ins to work properly for Android.
