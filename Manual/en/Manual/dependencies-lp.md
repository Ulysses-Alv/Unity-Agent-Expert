# Dependencies

The concept of dependencies is fundamental to Unity’s Package Manager behavior.

A dependency is a package that your project or another package needs to work.

| **Topic** | **Description** |
| --- | --- |
| **[Package dependency and resolution](upm-dependencies.md)** | Learn about the different types of dependencies and how Package Manager resolves them. |
| **[Package version conflict and resolution](upm-conflicts.md)** | Learn how Package Manager resolves package version conflicts when computing a compatible set of packages. |
| **[Embedded dependencies](upm-embed.md)** | Embed a package in your project to make it **mutable**. |
| **[Lock files](upm-conflicts-auto.md)** | Learn about lock files, which Package Manager uses to store the results of dependency resolution for a project. |
| **[Local folder or tarball paths](upm-localpath.md)** | Specify a dependency as a local folder or tarball that contains a package. |
