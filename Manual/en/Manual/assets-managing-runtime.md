# Managing assets at runtime

Unity has several options for managing and loading assets at runtime, including dynamically loading assets with AssetBundles, or the Addressables package to reduce the size of your build and distribute content updates.

| **Topic** | **Description** |
| --- | --- |
| **[Introduction to asset management](assets-managing-introduction.md)** | Understand Unity’s asset management options. |
| **[Direct reference asset management](assets-direct-reference.md)** | Understand how Unity builds and manages assets that you directly reference from your code. |
| **[Use the Resources system to load assets at runtime](assets-resources-system.md)** | Use a `Resources` folder to store assets that you can later retrieve and load at runtime using the associated [`Resources`](../ScriptReference/Resources.md) API. |
| **[Use AssetBundles to load assets at runtime](assetbundles-section.md)** | Organize and build assets into the AssetBundle archive format. |
| **[Avoiding asset duplication](assets-avoid-duplication.md)** | Understand when Unity duplicates assets and how to avoid or minimize this. |
