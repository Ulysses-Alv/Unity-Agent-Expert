# Built-in 3D physics

Unity’s default built-in 3D physics system is an integration of the Nvidia PhysX engine. You can use this system in object-oriented 3D projects.

This section includes the following topics.

| **Topic** | **Description** |
| --- | --- |
| **[Character control](character-control-section.md)** | Configure physics-based character control for first-person and third-person characters. |
| **[Rigidbody physics](rigidbody-physics-section.md)** | Apply physics-based behavior to **GameObjects**. |
| **[Collision](collision-section.md)** | Use **colliders** to configure collisions between GameObjects. |
| **[Joints](joints-section.md)** | Apply and configure joints that connect GameObjects and simulate physical forces for pivoting, movement, and restriction. |
| **[Articulations](articulations-section.md)** | Configure complex systems of rigid bodies and joints. |
| **[Ragdoll physics](ragdoll-physics-section.md)** | Configure ragdoll physics for characters. |
| **[Optimize physics performance](physics-optimization.md)** | Optimize physics system performance in the Unity Editor. |
| **[Cloth](class-Cloth.md)** | Simulate fabric movement for character clothing and other in-application textiles. |
| **[Multi-scene physics](physics-multi-scene.md)** | Manage different physics contexts in one project with multiple dedicated physics **scenes**. |
| **[Physics Debug window reference](PhysicsDebugVisualization.md)** | Use the Physics Debug window to configure and generate visual representations of 3D physics activity in a scene. |
| **[Physics Profiler module](ProfilerPhysics.md)** | Analyze the performance of physics in your application. |
