# Get started with 3D game development

**Tip:** If you have never used the Unity Editor before, start with the [First time users](first-time-user.md) documentation.

Everything in the Unity documentation, other than the 2D section, is relevant to 3D game development.

## Select platforms and render pipelines

The first two choices you must make are your 3D **render pipeline** and the platforms you want to support. These choices affect each other: some platforms don’t support the High Definition Render Pipeline (HDRP).

To get started, review the documentation for:

* [Platforms](PlatformSpecific.md)
* [Choosing a render pipeline](choose-a-render-pipeline-landing.md)

## Check third-party application compatibility

If you need to import assets from third-party applications, check that all applications can produce formats that Unity can import. For more information, refer to [Supported asset type reference](assets-supported-types.md).

## Create the game

What you focus on next depends on your role, but if you’re a sole creator, you might want to consider the following:

1. Create an environment in your game:
   * For a natural environment, use the [Terrain system](script-Terrain.md) or import an environment from a third-party modeling application.
   * For a built environment, such as buildings and rooms, you can start with [placeholders](PrimitiveObjects.md) or import [models](models.md).
2. Use [meshes](mesh.md) to place objects in the environment.
3. Change [lights](LightingOverview.md) and [visual effects](visual-effects.md) to create the right mood for your game.
4. [Animate](AnimationSection.md) characters and objects.
5. Create user [input controls](Input.md) and user [interfaces](UIToolkits.md).
